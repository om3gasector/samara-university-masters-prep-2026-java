[[Магистерский хаб]]

#### 8.1 Теория формальных языков как радел математической лингвистики. Понятие формального языка, грамматики. 
Формальный язык — это множество цепочек над конечным алфавитом. Теория формальных языков изучает способы задания языков через порождающие грамматики и распознающие автоматы. Формальная грамматика — это четверка (N, T, P, S): нетерминалы (синтаксические переменные), терминалы (алфавит языка), правила вывода (продукции), начальный символ (аксиома). Вывод — последовательная замена нетерминалов по правилам до получения терминальной цепочки. Язык грамматики — все терминальные цепочки, выводимые из аксиомы.
##### 8.1-1 ==Теория формальных языков как раздел математической лингвистики==
Теория формальных языков — это раздел дискретной математики и теоретической информатики, изучающий способы описания, порождения и распознавания множеств цепочек символов (языков) с помощью формальных систем (грамматик и автоматов).
	Возникновение теории связано с именами Н. Хомского (конец 1950-х годов), который предложил классификацию грамматик для описания естественных языков, и Дж. Бэкуса и П. Наура, разработавших форму записи синтаксиса языков программирования (БНФ). Оказалось, что разработанный для лингвистических целей аппарат идеально подходит для описания синтаксиса языков программирования, что и определило центральное место теории формальных языков в computer science.

**Место в системе наук:**
- Теория формальных языков является теоретическим фундаментом для построения трансляторов (компиляторов и интерпретаторов), текстовых анализаторов, систем искусственного интеллекта в части обработки естественного языка.
- Математическая база: теория множеств, теория графов, алгебра, математическая логика.
- Связь с теорией алгоритмов: каждому классу грамматик соответствует класс автоматов, способных распознавать языки этого класса.
##### 8.1-2 ==Понятие формального языка==
**Алфавит** — конечное непустое множество символов. Обозначается обычно _Σ_ (сигма) или _T_ (терминальный алфавит). Примеры: _Σ = {0, 1}_, _Σ = {a, b, ..., z}_.

**Цепочка (слово, строка)** — конечная последовательность символов алфавита. Длина цепочки _ω_ обозначается _|ω|_. Цепочка нулевой длины называется пустой цепочкой и обозначается _ε_ (эпсилон). Множество всех цепочек над алфавитом _Σ_, включая _ε_, обозначается Σ.

**Формальный язык** — произвольное множество цепочек над некоторым алфавитом: L ⊆ Σ. Язык может быть конечным или бесконечным.

**Способы задания формального языка:**
- **Перечисление** — применимо только для конечных языков. Пример: _L = {00, 01, 10, 11}
- **Порождающий способ (грамматики)** — задается множество правил, позволяющих породить все цепочки языка и только их.
- **Распознающий способ (автоматы)** — задается алгоритм (устройство), которое для любой цепочки определяет, принадлежит ли она языку.
- **Регулярные выражения, алгебраические уравнения** — декларативные формы описания.
##### 8.1-3 ==Понятие грамматики==
**Формальная порождающая грамматика G** — это четверка:

G=(N,T,P,S)

где:
- **N** — конечное множество нетерминальных символов (нетерминалов). Обозначаются обычно заглавными буквами латиницы. Синтаксические переменные, обозначающие промежуточные понятия языка.
- **T** — конечное множество терминальных символов (терминалов). Обозначаются строчными буквами или конкретными знаками. Это алфавит, над которым строится язык. Выполняется условие: _N ∩ T = ∅_.
- **P** — конечное множество правил вывода (продукций). Правило имеет вид _α → β_, где _α_ и _β_ — цепочки в объединенном алфавите (N ∪ T), причем _α_ содержит хотя бы один нетерминал. Читается: «α заменяется на β» или «из α непосредственно выводится β».
- **S** — начальный нетерминал (аксиома), _S ∈ N_. С него начинается порождение любой цепочки языка.
#### 8.2 Классификация грамматик по Хомскому. Техника построения автоматных и контекстно-свободных грамматик. 
Иерархия Хомского: тип 0 (неограниченные, машина Тьюринга), тип 1 (контекстно-зависимые, линейно-ограниченный автомат), тип 2 (контекстно-свободные, автомат с магазинной памятью), тип 3 (автоматные/регулярные, конечный автомат). Каждый следующий тип — частный случай предыдущего. Автоматные грамматики строятся по конечному автомату (состояния → нетерминалы, переходы → правила). КС-грамматики строятся с использованием операций альтернативы, конкатенации и рекурсии, позволяющих описывать вложенные (скобочные) конструкции.
##### 8.2-1 ==Классификация грамматик по Хомскому==
Ноам Хомский в 1956-1959 годах предложил иерархию формальных грамматик, основанную на постепенном сужении вида допустимых правил вывода. Эта классификация является фундаментом всей теории формальных языков. Выделяются четыре типа грамматик (от 0 до 3), каждый следующий тип является частным случаем предыдущего.

**Тип 0. Неограниченные грамматики (грамматики общего вида)**

Правила вывода имеют произвольный вид: _α → β_, где _α_ содержит хотя бы один нетерминал, _β_ — произвольная цепочка в _(N ∪ T)_, включая пустую. Никаких дополнительных ограничений нет.
Распознающий автомат: **машина Тьюринга**. Языки типа 0 называются **рекурсивно перечислимыми**. Это самый широкий класс; он включает все языки, которые могут быть порождены каким-либо алгоритмом. 

**Тип 2. Контекстно-свободные грамматики (КС-грамматики)**

Правила вывода имеют вид: _A → β_, где _A ∈ N_, _β ∈ (N ∪ T)_. Нетерминал заменяется независимо от контекста (символов, окружающих его в сентенциальной форме). Отсюда название — «контекстно-свободные».

Распознающий автомат: **автомат с магазинной памятью (МП-автомат)**. КС-грамматики являются основой для описания синтаксиса языков программирования: конструкции типа «выражение», «оператор», «блок» естественно описываются КС-правилами.

**Тип 3. Автоматные (регулярные) грамматики**
Правила вывода имеют один из двух видов (две эквивалентные формы):

- **Праволинейная форма:** _A → aB_ или _A → a_, где _A, B ∈ N_, _a ∈ T ∪ {ε}_.
- **Леволинейная форма:** _A → Ba_ или _A → a_, где _A, B ∈ N_, _a ∈ T ∪ {ε}_.

Распознающий автомат: **конечный автомат**. Регулярные языки используются для описания лексем (идентификаторов, чисел, ключевых слов) в компиляторах, для задания протоколов обмена, простых шаблонов поиска.
##### 8.2-2 ==Техника построения автоматных грамматик==
Техника построения автоматных грамматик заключается в переводе правил регулярного (праволинейного) выражения в правила грамматики вида _A → aB_ или _A → a_. Она строго эквивалентна построению конечного автомата.
Построение включает следующие шаги: 
1. **Задайте алфавиты**: определите терминалы (символы языка) и нетерминалы (состояния).
2. **Преобразуйте переходы в правила**: если автомат переходит из состояния _A_ в состояние _B_ по символу _a_, создайте правило _A → aB_.
3. **Обозначьте конец слова**: для финального состояния запишите правило _A → a_. Она строго эквивалентна построению конечного автомата., где  _a_ — последний символ перехода.
4. **Добавьте ∈ переходы**: если финального перехода нет, а состояние допускающее, добавьте правило_A → ∈_.
##### 8.2-3 ==Техника построения контекстно-свободных грамматик==
КС-грамматики способны описывать вложенные (скобочные) конструкции, принципиально недоступные для автоматных грамматик. Ключевая идея — использование рекурсии нетерминалов, позволяющей порождать сбалансированные структуры.

**Основные приемы построения КС-грамматик:**

**Прием 1. Итерация (последовательное соединение).**  
Если язык состоит из последовательности конструкций типа A, за которой идет конструкция типа B: правило _S → A B_. Далее _A_ и _B_ раскрываются независимо.

**Прием 2. Альтернатива (выбор).**  
Если конструкция может быть одного типа или другого: _S → A | B_.

**Прием 3. Рекурсия (самовставление).**  
Позволяет порождать неограниченно вложенные структуры.

**Общая методика:**
1. Выделить синтаксические категории языка (нетерминалы).
2. Определить структуру каждой категории: из чего она может состоять.
3. Записать правила, используя альтернативу, последовательное соединение и рекурсию.
4. Проверить, что грамматика порождает все требуемые цепочки и не порождает лишних.
##### 8.2-4 ==Понятие автомата==
**Определение:** Конечный автомат — это математическая модель дискретного устройства с конечной памятью, которое последовательно считывает символы входной цепочки и на каждом шаге меняет свое внутреннее состояние в зависимости от прочитанного символа и текущего состояния. По окончании чтения цепочки автомат выносит решение: допустить цепочку (она принадлежит языку) или отвергнуть.

Автомат является распознавателем — он не порождает цепочки, а принимает или отвергает их. Это дуальная модель к грамматике: грамматика порождает язык, автомат распознает его.

**Для чего нужны автоматы в IT (контекст экзамена):**
- Лексический анализ в компиляторах: сканер строится на основе конечного автомата, распознающего ключевые слова, идентификаторы, числа.
- Протоколы обмена: конечный автомат моделирует состояния соединения.
- Поиск подстрок, регулярные выражения в языках программирования (механизм regex).
- Моделирование дискретных систем с памятью о предыстории.
#### 8.3 Автоматные грамматики и конечные автоматы. Алгоритм приведения конечных автоматов и автоматных грамматик к детерминированной форме. 
Автоматные грамматики и конечные автоматы задают один и тот же класс — регулярные языки. По грамматике строится НКА (нетерминалы → состояния, правила → переходы), по автомату — грамматика (состояния → нетерминалы, переходы → правила). Детерминизация — преобразование НКА в ДКА методом подмножеств: каждое состояние ДКА соответствует множеству состояний НКА. Обрабатываются все символы алфавита, финальные состояния — те, что содержат хотя бы одно финальное состояние НКА. Результат — ДКА без неоднозначности переходов.
##### 8.3-1 ==Автоматные грамматики и конечные автоматы==
**Автоматные грамматики**:
Это грамматики, все правила (продукции) которых имеют один из двух видов:
- **Праволинейные:** _A → aB_ или _A → a_
- Леволинейные: _A → Ba_ или _A → a_ 
Где A и B — нетерминальные символы, а a — терминальный символ.
**Важное ограничение:** грамматика должна быть либо полностью праволинейной, либо полностью леволинейной. Смешивание правил обоих видов в одной грамматике не допускается, так как это может вывести порождаемый язык за пределы класса автоматных (регулярных) языков.

**Конечные автоматы**:
Конечный автомат (КА) — это абстрактный вычислитель, который состоит из конечного множества состояний, входной ленты с символами и функции переходов. Различают:
- **ДКА (Детерминированный конечный автомат):** для каждого текущего состояния и входного символа есть только один строго определенный переход.
- **НКА (Недетерминированный конечный автомат):** из одного состояния по одному и тому же символу возможен переход сразу в несколько состояний или не быть ни одного.

**Связь между ними**:
Согласно фундаментальной теореме теории автоматов, класс языков, порождаемых автоматными грамматиками, в точности совпадает с классом языков, распознаваемых конечными автоматами. Это класс **регулярных языков**.
1. По любой праволинейной (или леволинейной) грамматике можно построить эквивалентный ей **НКА**, который будет распознавать тот же самый язык.
2. Любой **ДКА** или **НКА** можно преобразовать в эквивалентную ему праволинейную автоматную грамматику.
3. Любой **НКА** может быть преобразован в эквивалентный ему **ДКА** (алгоритм детерминизации).
Обе модели взаимозаменяемы и находят широкое применение при разработке лексических анализаторов компиляторов, систем поиска подстрок и проектировании простых устройств. 
##### 8.3-2 ==Алгоритм приведения конечных автоматов к детерминированной форме (детерминизация НКА)==
Недетерминированный конечный автомат (НКА) может иметь несколько переходов по одному символу из одного состояния, а также ε-переходы. Для практической реализации (например, лексического анализатора) удобнее детерминированный автомат (ДКА), в котором из каждого состояния по каждому символу есть ровно один переход. Алгоритм детерминизации (алгоритм Томпсона, метод подмножеств) преобразует произвольный НКА в эквивалентный ДКА.

**Идея алгоритма (метод подмножеств):** каждое состояние строящегося ДКА соответствует **подмножеству состояний** исходного НКА. ДКА моделирует все возможные пути, по которым НКА может пойти при чтении цепочки.

**Алгоритм (пошагово):**
1. **Устранение ε-переходов (ε-замыкание):**  
    Для любого подмножества состояний _P_ определяется _ε-closure(P)_ — все состояния, достижимые из _P_ только по ε-переходам, включая сами состояния _P_. Это множество состояний, в которых НКА может находиться, не прочитав ни одного символа.
2. **Построение начального состояния ДКА:**  
    _q_0' = ε-c({q_0})_, где _q_0_ — начальное состояние НКА.
3. **Построение переходов ДКА (итеративно):**  
    Для каждого сформированного подмножества _P_ и каждого входного символа *a*:
	- Вычисляем _next = ε-closure( δ(P, a) )_, где _δ(P, a)_ — все состояния НКА, в которые можно перейти из _P_ по символу *a*.
	- _next_ становится новым состоянием ДКА (если оно еще не было создано).
	- Переход ДКА: _δ'(P, a) = next_.
4. **Определение конечных состояний ДКА:**  
    Подмножество _P_ является конечным состоянием ДКА, если оно содержит хотя бы одно конечное состояние исходного НКА.

**Результат:** ДКА с не более чем _2^{|Q|}_ состояниями, допускающий в точности тот же язык, что и исходный НКА.

**Практическое значение:** ДКА обрабатывает каждый символ за O(1), без перебора вариантов. Это делает его пригодным для эффективной программной реализации лексических анализаторов.

##### 8.3-3 ==Алгоритм приведения автоматных грамматик к детерминированной форме==
Приведение автоматной (регулярной) грамматики к детерминированной форме осуществляется через построение эквивалентного Недетерминированного конечного автомата (НКА) с последующей детерминизацией по алгоритму Томпсона и обратным переводом ДКА в грамматику.

Алгоритм состоит из трех последовательных этапов.

**Этап 1. Построение НКА по грамматике**
	Любая автоматная грамматика эквивалентна некоторому конечному автомату.
	**Состояния:** Каждому нетерминалу грамматики соответствует состояние автомата. Добавляется одно новое конечное состояние _F_ (если оно еще не присутствует среди состояний).

**Переходы:**
- Правилу вида _A → aB_ (где *a* — терминал, _B_ — нетерминал) соответствует переход из состояния _A_ в состояние _B_ по символу *a*.
- Правилу вида _A → a_ соответствует переход из состояния _A_ в заключительное состояние _F_ по символу *a*.
- Правилу вида _A → ε_ соответствует добавление состояния _A_ в множество конечных состояний (или ε-переход в _F_).

Результат: НКА, распознающий тот же язык, что порождает исходная грамматика.

---

**Этап 2. Детерминизация (преобразование НКА в ДКА)**
Построение детерминированного конечного автомата (ДКА) осуществляется **методом подмножеств**.
- Начальным состоянием нового ДКА является множество, состоящее из начального символа (аксиомы) грамматики (с учетом ε-замыкания, если в НКА присутствуют ε-переходы).
- Для каждого текущего множества состояний и для каждого символа алфавита определяются все возможные следующие состояния. Полученное множество состояний становится новым состоянием ДКА.
- Процесс повторяется, пока не будут найдены все достижимые комбинации состояний.
- Состояния ДКА, содержащие исходное заключительное состояние _F_ (или любое конечное состояние НКА), становятся заключительными состояниями ДКА.

Результат: ДКА, эквивалентный исходному НКА.

---

**Этап 3. Построение детерминированной грамматики по ДКА**
После того как ДКА построен, его переходы конвертируются обратно в детерминированную (праволинейную) автоматную грамматику.
- Каждому состоянию ДКА ставится в соответствие нетерминал.
- Переходу из состояния _A_ в состояние _B_ по терминалу *a* соответствует правило _A → aB_.
- Если состояние является заключительным, добавляется правило _A → ε_.

Результат: детерминированная автоматная грамматика, эквивалентная исходной. Для любой пары (нетерминал, терминал) в ней существует не более одного правила, что гарантирует однозначность вывода.
#### 8.4 Эквивалентные преобразования контекстно-свободных грамматик. Исключение тупиковых правил из грамматик. 
Эквивалентные преобразования КС-грамматик — это упрощение грамматики без изменения порождаемого языка. Включают:
- удаление бесполезных нетерминалов (не порождают терминалов);
- удаление недостижимых нетерминалов (не выводятся из аксиомы);
- удаление ε-правил (если они не нужны);
- удаление цепных правил (A → B).

Тупиковые правила — это правила, которые не могут участвовать в выводе ни одной цепочки языка. Их исключение — обязательный этап подготовки грамматики к использованию в синтаксическом анализаторе.
##### 8.4-1 ==Что такое эквивалентные преобразования КС-грамматик==
**Суть:** Эквивалентные преобразования — это операции над грамматикой, которые меняют набор правил и нетерминалов, но сохраняют порождаемый язык. Цель — упростить грамматику, убрать из нее лишнее, сделать удобной для дальнейшего анализа (например, для построения синтаксического анализатора).

**Зачем это нужно:** Исходная грамматика, описывающая язык программирования или структуру данных, часто содержит избыточные или бесполезные правила. Перед тем как строить компилятор, грамматику «чистят».

**Основные виды преобразований:**
1. Удаление бесполезных нетерминалов (тех, которые не могут участвовать в порождении терминальных цепочек).
2. Удаление недостижимых нетерминалов (тех, до которых нельзя добраться от начального символа).
3. Удаление ε-правил (правил вида _A → ε_), кроме случая, когда пустая цепочка входит в язык.
4. Удаление цепных правил (правил вида _A → B_, где оба — нетерминалы).
##### 8.4-2 ==Бесполезные нетерминалы и их удаление==
Бесполезный нетерминал — это такой нетерминал, из которого невозможно вывести ни одной терминальной цепочки. Проще говоря: он «зависает», из него нельзя дойти до конца, до конкретных символов алфавита.

**Пример:** правило _A → Aa | Ba_, если из _B_ тоже ничего не выводится. _A_ бесконечно ссылается сам на себя и никогда не выдаст готовую строку.

**Алгоритм удаления (идея):**  
Сначала помечаем нетерминалы, которые могут породить терминальную цепочку (хотя бы одну). Это делается снизу вверх: сначала помечаются те, у кого есть правило, полностью состоящее из терминалов. Затем те, у кого в правой части стоят уже помеченные. Остальные — бесполезные, удаляются вместе со всеми правилами, где они встречаются.
##### 8.4-3 ==Недостижимые нетерминалы и их удаление==
Недостижимый нетерминал — это такой, до которого невозможно добраться от начального символа грамматики. Даже если из него можно вывести терминальные цепочки, он никогда не будет использован при порождении.

**Пример:** в грамматике есть правила _S → aB, C → c_. Нетерминал _C_ не появляется ни в одном правиле, достижимом от _S_. Он бесполезен для языка.

**Алгоритм удаления (идея):**  
Начинаем с начального символа _S_ и идем вниз по правилам, отмечая все нетерминалы, которые встречаются в правых частях. Те, которые ни разу не встретились — недостижимы. Удаляем их и их правила.

**Важный порядок:** сначала удаляют бесполезные нетерминалы (те, что не дают терминалов), затем — недостижимые. Если сделать наоборот, можно оставить в грамматике мусор.
##### 8.4-4 ==Исключение тупиковых правил==
Тупиковое правило — это правило, которое либо содержит бесполезный нетерминал, либо является частью «тупиковой ветки» вывода, которая никогда не приводит к терминальной цепочке. Другими словами, тупиковые правила — это те, которые не могут участвовать в выводе ни одной цепочки языка.

**Виды тупиковых правил:**

- Правила, в правой части которых есть нетерминал, из которого нельзя вывести терминалы.
- Правила, которые сами по себе не дают выхода на терминалы и лишь комбинируют бесполезные нетерминалы.
- Правила, которые недостижимы от начального символа.

**Процедура исключения (объединяет два предыдущих пункта):**
1. Строим множество **продуктивных** нетерминалов — тех, из которых можно вывести хотя бы одну терминальную цепочку. Идем снизу вверх: если в правой части правила все нетерминалы уже продуктивны, то и левая часть продуктивна. Повторяем, пока множество не перестанет расти.
2. Все нетерминалы, не вошедшие в это множество — **непродуктивные**. Удаляем все правила, в которых они встречаются (слева или справа).
3. Строим множество **достижимых** нетерминалов. Начинаем с _S_, добавляем все нетерминалы из правых частей правил для уже достижимых. Повторяем.
4. Все недостижимые нетерминалы и их правила удаляем.

**Результат:** грамматика без тупиковых правил. Каждый нетерминал может породить терминальную цепочку и достижим от начального символа. Грамматика становится «чистой», готовой к дальнейшим преобразованиям (например, к построению синтаксического анализатора).
#### 8.5 Теорема об общем виде контекстно-свободных грамматик и следствия из нее. 
Теорема об общем виде КС-грамматик утверждает: любая КС-грамматика может быть преобразована к эквивалентной форме без ε-правил (кроме S → ε, если нужно), без цепных правил (A → B) и без бесполезных символов. Алгоритм: сначала убрать ε-правила, затем цепные, затем бесполезные символы (сначала непродуктивные, потом недостижимые). Следствия: грамматика становится пригодной для синтаксического анализа, может быть приведена к нормальной форме Хомского или Грейбах, легко проверяется на пустоту языка.
##### 8.5-1 ==О чем говорит теорема об общем виде контекстно-свободных грамматик==
Теорема об общем виде КС-грамматик утверждает, что любой контекстно-свободный язык (кроме, возможно, содержащего пустую цепочку) может быть порожден грамматикой, правила которой имеют строго определенную, «чистую» структуру. Это не меняет язык, но делает грамматику пригодной для алгоритмического разбора.

Другими словами: какой бы запутанной ни была исходная грамматика, ее всегда можно привести к стандартному виду, удобному для компилятора.

Общий вид КС-грамматики подразумевает, что грамматика удовлетворяет трем условиям:

**1. Отсутствие ε-правил (кроме, возможно, S → ε).**  
Правила вида _A → ε_ (пустая цепочка) усложняют анализ, так как нетерминал может «исчезнуть» в любой момент. Их можно удалить, кроме случая, когда пустая цепочка принадлежит языку — тогда сохраняется только правило для начального символа _S → ε_, и _S_ не встречается в правых частях других правил.

**2. Отсутствие цепных правил.**  
Цепное правило — это правило вида _A → B_, где оба — нетерминалы. Такие правила создают лишние шаги вывода и циклы. Их заменяют: если _A → B_ и _B → α_, то добавляют _A → α_, а цепное правило удаляют.

**3. Отсутствие бесполезных символов.**  
Все нетерминалы должны быть продуктивными (из них можно вывести терминальную цепочку) и достижимыми (до них можно дойти от начального символа). 
##### 8.5-2 ==Следствия из теоремы об общем виде контекстно-свободных грамматик==

**Следствие 1. Возможность синтаксического анализа.**  
Грамматика, приведенная к общему виду, не содержит «ловушек» — правил, которые заводят анализ в тупик или бесконечный цикл. Это обязательное условие для работы любого алгоритма разбора (нисходящего или восходящего).

**Следствие 2. Нормальные формы.**  
Общий вид является основой для дальнейшей нормализации. Наиболее известные нормальные формы КС-грамматик:
- **Нормальная форма Хомского (НФХ):** все правила имеют вид _A → BC_ или _A → a_, где _A, B, C_ — нетерминалы, *a* — терминал. Удобна для алгоритма Кока-Янгера-Касами (CYK).
- **Нормальная форма Грейбах (НФГ):** все правила имеют вид _A → aα_, где *a* — терминал, _α_ — цепочка нетерминалов (возможно, пустая). Удобна для нисходящего разбора без возвратов.

Любая КС-грамматика в общем виде может быть преобразована в любую из этих форм.

**Следствие 3. Разрешимость проблемы пустоты.**  
Для грамматики в общем виде легко проверить, порождает ли она хотя бы одну цепочку: достаточно проверить, продуктивен ли начальный символ. Если да — язык не пуст.

**Следствие 4. Устранение неоднозначности (частичное).**  
Приведение к общему виду не гарантирует однозначности, но удаляет источники «паразитной» неоднозначности, связанные с ε-правилами и цепными правилами. Оставшаяся неоднозначность — уже свойство самого языка, а не дефект записи грамматики.
#### 8.6 Операции над языками. Замкнутость автоматных и контекстно-свободных грамматик относительно операций. 
Над формальными языками определены операции: объединение, конкатенация, итерация, пересечение, дополнение, разность, обращение. (подрбно в 7.6.1)

Автоматные (регулярные) языки замкнуты относительно всех перечисленных операций — результат применения операции к регулярным языкам снова является регулярным. 

Контекстно-свободные языки замкнуты относительно объединения, конкатенации, итерации и обращения, но **не замкнуты** относительно пересечения и дополнения. Это ключевое различие важно для определения границ применимости грамматик при построении компиляторов.
##### 8.6-1==Основные операции над языками==

Языки — это множества цепочек над алфавитом. Соответственно, к ним применимы теоретико-множественные операции и специальные лингвистические операции.

**Объединение** _L₁ ∪ L₂_ — множество цепочек, принадлежащих хотя бы одному из двух языков.

**Конкатенация (произведение)** _L₁ · L₂_ — множество цепочек, полученных приписыванием любой цепочки из _L₂_ в конец любой цепочки из _L₁_. Порядок важен.

**Итерация (звезда Клини)** *L** — множество всех цепочек, которые можно получить конкатенацией любого (в том числе нулевого) числа цепочек из _L_. Включает пустую цепочку ε.

**Пересечение** _L₁ ∩ L₂_ — множество цепочек, принадлежащих обоим языкам одновременно.

**Дополнение** _L̄_ — множество всех цепочек над алфавитом, не принадлежащих языку _L_. Зависит от объемлющего алфавита.

**Разность** _L₁ \ L₂_ — множество цепочек из _L₁_, не входящих в _L₂_.

**Обращение (зеркальное отражение)** _L^R_ — множество цепочек, записанных в обратном порядке.
##### 8.6-2 ==Замкнутость автоматных (регулярных) языков==

Автоматные языки замкнуты относительно **всех** перечисленных операций. Это означает, что если взять регулярные языки и применить к ним любую из операций, результат всегда будет регулярным языком. Это свойство доказывается конструктивно — через построение конечных автоматов для результата.

**Объединение, конкатенация, итерация:** строятся на основе исходных автоматов добавлением ε-переходов к новым начальным и конечным состояниям. Эти три операции — основа регулярных выражений.

**Пересечение:** строится автомат-произведение, состояния которого — пары состояний исходных автоматов. Конечные состояния — пары, где оба компонента конечные.

**Дополнение:** в детерминированном полном автомате все конечные состояния делаются неконечными, а неконечные — конечными. Это меняет решение на противоположное.

**Разность:** выражается через пересечение и дополнение: _L₁ \ L₂ = L₁ ∩ L̄₂_.

**Обращение:** в автомате меняется направление всех переходов, начальное и конечные состояния меняются ролями. Исходный автомат должен быть с единственным конечным состоянием (к нему всегда можно привести).
##### 8.6-3 ==Замкнутость контекстно-свободных языков==
**Замкнутость (операции, сохраняющие КС-свойство):**

- **Объединение:** если есть грамматики _G₁_ и _G₂_, создается новая грамматика с новым начальным символом _S_ и правилами _S → S₁ | S₂_, где _S₁_ и _S₂_ — начальные символы исходных грамматик. Язык сохраняется.
- **Конкатенация:** аналогично, новое правило _S → S₁ S₂_.
- **Итерация:** новое правило _S → S₁ S | ε_ (или _S → S S₁ | ε_). Обеспечивает любое количество повторений.
- **Обращение:** просто во всех правилах грамматики правые части записываются в обратном порядке. Легко доказывается, что порождается обращенный язык.

**Незамкнутость (операции, выводящие за пределы КС-класса):**

- **Пересечение:** существуют два КС-языка, пересечение которых не является КС. Классический пример: _L₁ = {aⁿ bⁿ cᵐ | n, m ≥ 1}_ и _L₂ = {aᵐ bⁿ cⁿ | n, m ≥ 1}_. Оба — КС, их пересечение _L₁ ∩ L₂ = {aⁿ bⁿ cⁿ | n ≥ 1}_ — известный пример не-КС языка. Следовательно, КС-языки не замкнуты по пересечению.
- **Дополнение:** если бы КС-языки были замкнуты по дополнению, то в силу замкнутости по объединению и формул де Моргана они были бы замкнуты и по пересечению. Но пересечение не сохраняется, значит, и дополнение не сохраняется. Поэтому КС-языки не замкнуты по дополнению.
- **Разность:** выражается через пересечение и дополнение, так что тоже не замкнуты.
##### 8.6-4 ==Практическое значение замкнутости==
Знание о замкнутости помогает разработчику компилятора понять, какие конструкции языка можно комбинировать, оставаясь в рамках того же класса анализаторов. Например, лексический анализатор (регулярный язык) можно как угодно комбинировать с помощью объединений и пересечений, не боясь выйти за возможности конечных автоматов. А для синтаксического анализатора (КС-язык) нельзя просто так взять пересечение двух грамматик и ожидать, что полученный язык будет разбираться теми же методами.
#### 8.7 Методы анализа контекстно-свободных грамматик. Грамматики предшествования Вирта и Флойда. 
Синтаксический анализ — это процесс определения, принадлежит ли входная цепочка языку, и построения дерева вывода. Методы делятся на нисходящие (от аксиомы к цепочке) и восходящие (от цепочки к аксиоме). Грамматики предшествования — класс КС-грамматик, для которых отношения между соседними символами в сентенциальной форме однозначно определяют, какой участок цепочки сворачивать. Отношения предшествования (меньше, равно, больше) задаются на основе вычисления множеств крайних левых и крайних правых символов. Метод Вирта-Флойда — это восходящий анализатор на основе матрицы предшествования, работающий по принципу «перенос-свертка» и находящий границы основы (сворачиваемой подцепочки) без полного перебора.
##### 8.7-2 ==Что такое синтаксический анализ==
Синтаксический анализ (парсинг) — второй этап работы компилятора после лексического анализа. Задача: получить от сканера поток лексем (токенов) и определить, соответствует ли их последовательность грамматике языка. Если соответствует — построить синтаксическое дерево (или цепочку вывода). Если нет — выдать сообщение об ошибке.

**Два основных направления анализа:**
- **Нисходящий (top-down):** построение дерева от корня (аксиомы) к листьям (терминалам). Методы: рекурсивный спуск, LL-анализ.
- **Восходящий (bottom-up):** построение дерева от листьев к корню. Методы: анализаторы предшествования, LR-анализ.
##### 8.7-3 ==Идея грамматик предшествования==
Грамматики предшествования основаны на идее, что решение о том, где в цепочке находится «основа» (подцепочка, которую нужно свернуть к нетерминалу), можно принять, глядя только на отношения между соседними символами. Это позволяет избежать полного перебора возможных сверток.

**Принцип работы анализатора:**  
Просматривая цепочку, анализатор ищет момент, когда отношение меняется с ⋖ на ⋗. Участок между этими границами — основа, которая сворачивается к нетерминалу. Процесс повторяется, пока вся цепочка не свернется к аксиоме.
##### 8.7-3 ==Грамматики Вирта и Флойда==
Никлаус Вирт и Роберт Флойд независимо развили идею предшествования в 1960-х годах. Их подходы немного различаются:
- **Флойд** ввел формализм отношений предшествования и алгоритм восходящего разбора для операторных грамматик (где нет двух нетерминалов подряд). Это называется **грамматиками операторного предшествования**.
- **Вирт** адаптировал этот метод для более широкого класса грамматик и встроил его в свои системы построения компиляторов (Pascal-подобные языки). Метод получил название **простого предшествования**.

**Алгоритм разбора:**
1. Входная цепочка лексем помещается в буфер с маркерами начала и конца.
2. Анализатор сканирует цепочку, сравнивая отношения между соседними символами по матрице.
3. Когда обнаруживается последовательность вида: _⋖ ... ≐ ... ⋗_, внутренняя часть сворачивается к нетерминалу согласно правилу грамматики.
4. Процесс повторяется до свертки к аксиоме или обнаружения ошибки (если отношение не определено).
#### 8.8 Функции предшествования. Методы построения функций предшествования. 
Функции предшествования — это способ компактного представления матрицы предшествования. Вместо хранения отношений для каждой пары символов, каждому символу грамматики сопоставляются два целых числа: _f(s)_ — функция левого контекста, _g(s)_ — функция правого контекста. Отношения предшествования определяются сравнением: _a ⋖ b_, если _f(a) < g(b)_; _a ≐ b_, если _f(a) = g(b)_; _a ⋗ b_, если _f(a) > g(b)_. Метод построения: по матрице предшествования строится ориентированный граф, вершины — символы грамматики в двух экземплярах (f и g), дуги — отражение отношений. Функции вычисляются как длины кратчайших путей в этом графе. Если граф содержит циклы, функции предшествования для данной грамматики не существуют.
##### 8.8-1 ==Что такое функции предшествования==
Функции предшествования — это альтернативный способ задания отношений предшествования без явного хранения матрицы размером _n × n_. Вместо хранения одного из трех значений для каждой пары символов, каждому символу *s* (терминалу или нетерминалу) ставятся в соответствие два целых числа:
- **f(s)** — функция предшествования слева. Характеризует поведение символа, когда он находится слева от другого символа.
- **g(s)** — функция предшествования справа. Характеризует поведение символа, когда он находится справа от другого символа.

Отношения предшествования между символами *a* и *b* определяются простым сравнением этих чисел:
- _a ⋖ b_, если _f(a) < g(b)_ — основа начинается правее *a*.
- _a ≐ b_, если _f(a) = g(b)_ — *a* и *b* принадлежат одной основе.
- _a ⋗ b_, если _f(a) > g(b)_ — основа заканчивается на *a*.

Преимущество: вместо матрицы размером _n × n_ хранится всего _2n_ чисел. Это значительно экономит память и упрощает программную реализацию. Недостаток: функции предшествования существуют не для всех грамматик предшествования, а только для тех, у которых матрица предшествования не содержит конфликтных циклов.
##### 8.8-2 ==Метод построения функций предшествования==
Функции предшествования f (левый контекст) и g (правый контекст) строятся по матрице предшествования через ориентированный граф. Для каждого символа создаются две вершины: f_s и g_s. Дуги проводятся по отношениям: если a ≐ b, то f_a и g_b связываются в обе стороны (равенство); если a ⋖ b, то дуга из g_b в f_a (g(b) должно быть больше f(a)); если a ⋗ b, то дуга из f_a в g_b (f(a) должно быть больше g(b)). Значения функций — максимальные длины путей из соответствующих вершин. Если граф содержит циклы, функции предшествования для данной грамматики не существуют, и необходимо использовать полную матрицу.
**Простыми словами:**
	У нас есть таблица (матрица), в которой для каждой пары символов записано, кто кого главнее: левый символ меньше правого, равен ему или больше.
	.
	Мы хотим вместо большой таблицы получить две небольшие числовые линейки: одну для символов слева (f), другую для символов справа (g). Числа подбираются так, чтобы при сравнении они давали те же отношения, что и в таблице: если левый меньше — его число меньше, если равны — числа равны, если левый больше — его число больше.

**Как подбирают числа:**  
Рисуют граф, где каждый символ представлен дважды: один раз как левый, другой раз как правый. Затем по таблице проводят стрелки.
- Если символы равны по таблице — стрелки в обе стороны (их числа должны совпадать).
- Если левый меньше правого — стрелка от правого к левому (число правого должно выйти больше).
- Если левый больше правого — стрелка от левого к правому (число левого должно выйти больше).

Число для каждого символа — это самая длинная цепочка стрелок, которая из него выходит. Длиннее путь — больше число.
#### 8.9 Польская инверсная запись (Полиз). Интерпретация Полиза. Алгоритм Заммельсона и Бауэра перевода выражений в Полиз.
ПОЛИЗ (польская инверсная запись, постфиксная нотация) — форма записи выражений, в которой знак операции записывается после операндов. Не требует скобок и правил приоритета: порядок вычислений однозначно определяется порядком следования знаков. Интерпретация ПОЛИЗа выполняется стековой машиной: операнды помещаются в стек, операция извлекает нужное число операндов, выполняет действие и помещает результат обратно. Алгоритм Замельсона-Бауэра переводит инфиксное выражение в ПОЛИЗ за один проход с использованием стека операций: операнды сразу выводятся, операции помещаются в стек с учетом приоритетов, скобки управляют выталкиванием.
##### 8.9-1 ==Польская инверсная запись (Полиз)==
Польская инверсная запись — это способ записи арифметических и логических выражений, при котором операторы следуют за своими операндами. Отсюда второе название: постфиксная нотация. Предложена польским логиком Яном Лукасевичем.

**Пример:** инфиксное выражение _(a + b) * c_ в ПОЛИЗе записывается как _a b + c *_.

**Свойства ПОЛИЗа:**
- Скобки полностью отсутствуют. Приоритет операций и порядок вычислений закодированы в порядке следования знаков операций.
- Выражение читается строго слева направо.
- Каждая операция применяется к нужному числу непосредственно предшествующих операндов.
- Никакой неоднозначности: одно выражение — ровно одна последовательность вычислений.

ПОЛИЗ широко применяется в компиляторах как промежуточное представление программы перед генерацией машинного кода, потому что он максимально приближен к стековой модели вычислений.
##### 8.9-2 ==Интерпретация Полиза==
Интерпретация (вычисление) выражения в ПОЛИЗе выполняется с помощью стека. Алгоритм прост:

1. Просматриваем ПОЛИЗ слева направо, символ за символом.
2. Если очередной элемент — операнд (число, переменная, идентификатор): помещаем его значение в стек
3. Если очередной элемент — знак операции: извлекаем из стека нужное число операндов (два для бинарных операций, один для унарных), выполняем операцию, результат помещаем обратно в стек.
4. По окончании просмотра в стеке остается ровно одно значение — результат вычисления всего выражения.

**Пример вычисления:** ПОЛИЗ _a b + c *_.

- *a* → стек: \[a]
- *b* → стек: \[a, b]
- *+* → извлекли b, a, сложили → стек: \[a+b]
- *c* → стек: \[a+b, c]
- *** → извлекли c, a+b, умножили → стек: \[(a+b)\*c]

Конец. Результат: \_(a+b)\_c*.
##### 8.9-3 ==Алгоритм Заммельсона и Бауэра перевода выражений в Полиз==
Алгоритм Замельсона-Бауэра (также известен как алгоритм «сортировочной станции» Э. Дейкстры) переводит выражение из инфиксной формы в ПОЛИЗ за один проход. Используется входная строка, выходная строка (формируемый ПОЛИЗ) и стек для временного хранения знаков операций.

**Вход:** инфиксное выражение с операндами, бинарными операциями и скобками.  
**Выход:** эквивалентное выражение в ПОЛИЗе.

**Алгоритм:**
1. Просматриваем входное выражение слева направо.
2. Если встретился операнд — сразу записываем его в выходную строку.
3. Если встретилась открывающая скобка — помещаем ее в стек.
4. Если встретилась закрывающая скобка — выталкиваем из стека в выходную строку все знаки операций до ближайшей открывающей скобки. Открывающую скобку удаляем из стека (в выход не пишем).
5. Если встретился знак операции — выталкиваем из стека в выходную строку все знаки операций, имеющие приоритет **выше или равный** текущему (с учетом левой ассоциативности). Затем помещаем текущий знак в стек
6. По окончании входной строки выталкиваем из стека все оставшиеся знаки операций в выходную строку.

**Пример:** _a + b * c_ → ПОЛИЗ _a b c * +_.
**Почему алгоритм важен:** это классический и простой в реализации метод, используемый в учебных компиляторах и интерпретаторах. Демонстрирует применение стека для задач синтаксического анализа и трансляции.
